Interrupting

Or how to catch back if you're drowning.

Let's say you posted so many tasks to your threadpool that all your cores are full, still, your application is slipping more and more behind plan. You need to give up some tasks to catch back a little.

Asynchronous can give you an interruptible cookie when you post a task to a scheduler, and you can use it to stop a posted task. If not running yet, the task will not start, if running, it will stop at the next interruption point, which are documented in the Boost.Thread documentation. Diagnostics will show that a task was interrupted.
